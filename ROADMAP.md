# TimeComplex 开发路线图与需求清单 (Roadmap & To-Do)

本文档记录了 TimeComplex 项目目前存在的问题、需要完善的功能以及中远期目标。
在没有新的具体需求时，可以主动浏览此清单，根据优先级逐步完成任务。

## ✨ 一、目前存在的问题 (待修复 / 待实现)

### 部分数据尚未对接后端数据库
- [ ] **时间块 (TimeBlock)**：目前创建的区间方块数据只保存在前端的 React State 中，刷新页面就会丢失。代码中预留了 `// TODO: 实现后端 API 支持 TimeBlock`，需要完善后端的 `/api/timeblocks` 路由以实现云端持久化。 **（高优先级）**
- [ ] **涂鸦和橡皮擦数据 (Strokes)**：刚加入的画板数据目前也只是存在内存中，没有与用户账号绑定，刷新后画笔轨迹和擦除记录都会消失，需要设计一种轻量级的方式（例如保存为 SVG 数据系或 JSON 格式）存入数据库。 **（中优先级）**

### 工具状态管理冗余
- [ ] 当切换不同工具（画笔、事件点、方块）时，各种 `pendingXXX` 和 `editingXXX` 的状态清理逻辑耦合在了一起。如果后续工具变多，这部分代码会越来越臃肿，容易产生边界 Bug（比如正在画画的时候突然触发了事件点击）。 **（高优先级）**

### 标签贴纸 (Sticker) 工具未实现
- [ ] 工具栏里虽然有“标签贴纸”的按钮，但点击后在画布上并没有对应的交互逻辑，仅仅是切换了高亮状态，功能是缺失的。 **（低优先级）**

### 移动端适配较差
- [ ] 目前事件主要依赖于 `onMouseDown/Move/Up` 鼠标事件，虽然做了防误触的 `touch-none`，但缺乏针对移动端原生 `onTouchStart/Move/End` 的专门处理。如果用手机打开，无论是创建节点还是画画，体验都会比较生硬。 **（低优先级）**

## ✨ 二、建议完善与优化的功能 (Next Steps)

### 项目化管理与“多维时间线”
如果能引入 **“项目 (Project)”** 或 **“图层 (Layer)”** 的概念，体验会更上一层楼：
- [ ] **分层展示**：允许用户创建不同的域（Realm）（例如：域 A 是“个人成长”，域 B 是“行业技术演进”），可以一键切换显示或重叠对比。 **（高优先级）**
- [ ] **时间线对比**：支持同时打开两条时间线。例如，将“公司融资历程”与“宏观经济波动”并排对比，发现其中的关联性。 **（中优先级）**

### 时间线，事件关系的连线连结
- [ ] **增加“时间线”功能**：用户可以选中两个目标（可以是时间块，也可以是事件点），画出一条“关联线”，表示因果关系（如：事件 A 导致了 事件 B），它们会被属于时间线 xxx。 **（中优先级）**

### “时间块”功能的设计与完善（核心功能）
- [ ] **优化操作**：在“时间块”（统一叫这个名字）模式下，可以直接拖动时间块的左边线和右边线改变其对应的时间；简化编辑模式，标题直接在方块上编辑，编辑弹窗只保留时间和颜色选择。 **（高优先级）**

### 性能优化：画布虚拟化 (Virtualization)
- [ ] 如果时间轴上有几千个事件点或复杂的画笔涂鸦，使用 React 渲染庞大的 `<svg>` 节点会导致严重的卡顿。可以实现基于当前 `offset` (可视区域) 的视口裁剪，只渲染屏幕内的元素，屏幕外的元素不进行 DOM 挂载。 **（高优先级，不影响程序的主要功能的状态下开发）**

## ✨ 三、中期目标

### AI 语义关联与知识图谱
可以不局限于“提取事件”，而是进一步挖掘数据价值：
- [ ] **时间线自动生成**：利用 AI 分析画布上已有的多个事件，自动生成连接线（Line/Arrow），标注“导致”、“触发”或“相关”等关系。 **（高优先级）**
- [ ] **智能填充**：当用户在一个区块（Block）内打点时，AI 可以根据上下文推荐相关的历史背景事件，帮助用户补完背景。 **（中优先级）**

### 🔗 协作与分享生态
- [ ] **公共广场 (Discover)**：允许用户将自己的时间轴设为“公开”，其他用户可以像刷博客一样查看别人整理的高质量时间线（如《指环王编年史》或《SpaceX 发射史》）。 **（低优先级）**
- [ ] **一键导出**：支持将时间轴导出为高分辨率长图、PDF，甚至是非线性叙事的 PPT 演示模式。 **（低优先级）**
